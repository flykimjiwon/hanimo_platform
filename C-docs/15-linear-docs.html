hanimo · open-source AI coding platform

The AI coding agent
built for the open web.

hanimo is a Go-based terminal AI coding agent. Successor to Modol. 14+ LLM providers, hash-anchored editing for zero hallucinated changes, local-first architecture, and a single ~20 MB binary with no runtime dependencies.

capabilities

Core capabilities

hanimo is built around six technical capabilities that distinguish it from every other AI coding tool. Each is designed for engineers who care about correctness, not just convenience.

Unique differentiator

Hash-Anchored Editing is hanimo's most distinctive capability. No other AI coding tool verifies edits at the line level before applying them.

01 — Unique
Hash-Anchored Editing
MD5 hash computed per line before any edit. Each proposed change verified against stored hash. Mismatches rejected. Zero hallucinated edits to wrong lines.
02 — Modes
Super / Dev / Plan
Three operating modes via Tab. Super for full autonomy. Dev for interactive pair-programming. Plan for structured multi-step execution with approval gates.
03 — Autonomy
/auto Autonomous Mode
Up to 20 autonomous iterations. Reads code, applies hash-verified edits, runs tests, fixes failures, repeats. Configurable with --max-iter=N.
04 — Persistence
SQLite Sessions
Every conversation persisted to SQLite. /session save, load, fork, and full-text search. Resume exactly where you left off.
05 — Context
3-Stage Compaction
Summarize at 80%, compress at 90%, evict at 95%. Never hit a context wall mid-task. Long sessions stay sharp and coherent automatically.
06 — Extensibility
MCP stdio + 14 Tools
Full Model Context Protocol via stdio. 14 built-in tools: file, shell, browser, code analysis. Ctrl+K command palette for instant access.

Configuration reference

hanimo is configured via config.yaml in your home or project directory. The .hanimo.md file in any project directory sets project-specific context.

~/.config/hanimo/config.yaml yaml
# Core provider settings provider: anthropic model: claude-3-5-sonnet-20241022 theme: honey # honey|ocean|dracula|nord|forest # Edit strategy edit: strategy: hash-anchored # unique to hanimo verify: md5 strict: true # Context management context: compaction: 3-stage summarize_at: 0.80 compress_at: 0.90 evict_at: 0.95 # MCP servers mcp_servers: postgres: cmd: mcp-postgres args: ["$DATABASE_URL"] github: cmd: mcp-github # Session storage session: backend: sqlite auto_save: true
./myproject/.hanimo.md · project context markdown
# Project: myapp language: Go test_cmd: go test ./... lint_cmd: golangci-lint run build_cmd: go build ./cmd/myapp # Project-specific context for the AI agent notes: | Auth module uses JWT with RS256. Database: PostgreSQL via pgx/v5. API style: REST, OpenAPI 3.1 spec in ./api/.

installation

Installation

hanimo is distributed as a single statically-linked binary. No runtime dependencies. Three install methods available.

# Install latest stable binary $ curl -fsSL https://hanimo.dev/install.sh | sh # Set your provider API key $ export ANTHROPIC_API_KEY=sk-ant-... # Launch $ hanimo
# Tap and install $ brew tap flykimjiwon/hanimo $ brew install hanimo # Set API key $ export ANTHROPIC_API_KEY=sk-ant-... $ hanimo
# Requires Go 1.22+ $ git clone https://github.com/flykimjiwon/hanimo $ cd hanimo $ go build -o hanimo ./cmd/hanimo $ sudo mv hanimo /usr/local/bin/ $ export ANTHROPIC_API_KEY=sk-ant-... $ hanimo

First run

  1. Install the binary
    Run one of the install commands above. The binary is ~20 MB with no runtime dependencies.
  2. Set your provider API key
    Export ANTHROPIC_API_KEY, OPENAI_API_KEY, or any other supported provider key. Or use Ollama for fully local inference — no API key required.
  3. Run hanimo in your project directory
    hanimo auto-detects your project structure and loads .hanimo.md if present. Start with /auto to delegate your first task.
