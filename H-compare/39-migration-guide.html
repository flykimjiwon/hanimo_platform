1
제거
Cursor 제거 (선택 사항)
hanimo는 터미널 기반입니다. Cursor를 유지해도 충돌 없음.
.cursorrules 파일은 그대로 두세요. 다음 단계에서 변환합니다.
2
설치
hanimo 설치
단일 Go 바이너리. 런타임 불필요.
클릭하여 복사
$ curl -sSL https://hanimo.dev/install | sh
# 또는
$ brew install hanimo
# 또는
$ go install github.com/flykimjiwon/hanimo@latest
# ~20MB, 의존성 없음
3
설정
모델 연결
기존 API 키 재사용 또는 Ollama 로컬 실행.
클릭하여 복사
# Anthropic 사용 (기존 키 재활용)
$ hanimo config set provider anthropic
$ hanimo config set api-key $ANTHROPIC_API_KEY

# 또는 Ollama로 완전 로컬 ($0/월)
$ hanimo config set provider ollama
$ hanimo config set model llama3.2
4
이전
.cursorrules → .hanimo.md 변환
Cursor의 규칙 파일을 그대로 가져올 수 있습니다.
클릭하여 복사
$ cp .cursorrules .hanimo.md
# 끝입니다. 형식 변환 없음.
Cursor 기능 → hanimo 대응 비고
.cursorrules→.hanimo.md 직접 복사
Cmd+K 인라인→ --mode dev 인터랙티브
Composer (에이전트)→ --mode super 자율 실행
Cursor 채팅→ --mode plan 계획 전용
@파일 참조→ --context 파일·폴더
세션 재시작→ hanimo session resume SQLite 영구
